# Инициализация Supabase для Way Clans

## Проблема
В Supabase отсутствуют таблицы базы данных. Ресурсы и данные не загружаются из БД.

## Решение

### Шаг 1: Откройте SQL Editor в Supabase
1. Перейдите в ваш проект Supabase (https://supabase.com)
2. Откройте вкладку **SQL Editor**
3. Нажмите **New Query**

### Шаг 2: Скопируйте и запустите SQL скрипт
1. Откройте файл `supabase-init.sql` из корня проекта
2. Скопируйте весь контент
3. Вставьте в SQL Editor Supabase
4. Нажмите **Run** (зелёная кнопка с "▶")

### Шаг 3: Проверьте результат
После выполнения скрипта должны появиться таблицы:
- ✅ **users** - данные игроков
- ✅ **buildings** - постройки
- ✅ **building_configs** - конфигурации типов построек
- ✅ **production_logs** - логи производства

### Шаг 4: Проверьте в МiniApp
- Откройте MiniApp
- Нажмите команду `/start` в Telegram боте
- Пользователь должен создаться в таблице `users`
- Должны создаться 4 начальные постройки (Шахта, Каменоломня, Лесопилка, Ферма)

---

## Что делает скрипт?

1. **Создаёт таблицы**:
   - `users` - профили игроков с ресурсами
   - `buildings` - постройки игроков
   - `building_configs` - типы и параметры построек
   - `production_logs` - логирование производства

2. **Вставляет конфигурации построек**:
   - ⛏ Шахта (производит золото)
   - ⛏ Каменоломня (производит камень)
   - 🌲 Лесопилка (производит дерево)
   - 🍖 Ферма (производит мясо)

3. **Включает RLS** (Row Level Security) для защиты данных

---

## Если что-то пошло не так?

### Ошибка: "relation "users" already exists"
- Это означает что таблица уже создана
- Просто игнорируйте эту ошибку или удалите существующие таблицы и запустите скрипт заново

### Ошибка при удалении таблиц
Используйте этот скрипт для очистки:
```sql
DROP TABLE IF EXISTS production_logs CASCADE;
DROP TABLE IF EXISTS buildings CASCADE;
DROP TABLE IF EXISTS building_configs CASCADE;
DROP TABLE IF EXISTS users CASCADE;
```

После этого запустите `supabase-init.sql` заново.
